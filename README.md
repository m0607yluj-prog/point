# きっずポイント

親子で使う「問題作成・採点・ポイント管理・ご褒美管理」アプリです。

## できること

**保護者**
- 問題を作成（選択式・記述式）
  - 複数行のテキストを貼り付けて記述式問題を一括登録（番号・丸数字は自動で除去）
  - CSV（教科,単元,難易度,問題,解答,解説）を貼り付けて一括登録（Excel等からのコピー＆ペーストにも対応）
  - 問題ごとに期限・期限切れ減点・対象の子どもを設定可能（期限を過ぎて未回答だと自動で減点）
  - 教科・単元・難易度・期限で登録済みの問題を絞り込み表示
  - 期限が今日の問題があると「問題作成」タブが赤く強調表示、3日以内ならうっすら強調表示される
- 記述式の回答を採点（CSVの「解答」「解説」列は採点時の参考として表示。自動採点はしません）
- 不正解の問題は自動で2日後期限・同じ減点設定でその子専用に再登録（正解するまで繰り返し）
- お手伝いを登録（定型・随時）し、完了報告を承認してポイント付与
- 勉強タスクを登録（画面から追加、またはCSV「教科,単元,内容,ポイント」で一括登録）
- お手伝い・勉強タスクには「達成度レベル」（例: かんぺき20P/よくできた10P/もうすこし3P）を自由に設定可能。設定した場合、承認時にレベルを選んでポイントを付与
- ポイントを手動で増減
- ご褒美を登録（必要ポイント・在庫）
- こどもを複数人登録

**こども**
- 自分のプロフィールを選んで問題に回答
- 選択式は即採点、記述式は保護者の採点待ち
- 教科・単元・難易度・期限で問題を絞り込んで表示
- 期限つきの問題は期限も表示され、過ぎると自動で減点（履歴に「期限切れ」として表示）
- 期限が今日の問題があると「もんだい」タブが赤く強調表示、3日以内ならうっすら強調表示される
- 不正解だった問題は2日後の期限つきで自動的に再挑戦できるようになる
- 採点後に解説（設定されていれば）を確認できる
- お手伝い・勉強タスクを「やった！」と報告し、保護者の承認でポイント獲得（タブは別々）
  - 定型（いつでもできる）：1日1回まで報告可能
  - 随時（保護者からの依頼）：割り当てられた子ども（または誰でも）が1回だけ報告可能
  - 達成度レベルが設定されている場合は、承認時に選ばれたレベルに応じたポイントが履歴に表示される
- 結果（履歴）の確認
- ポイントでご褒美と交換

## データの保存先

データはPostgresデータベースに保存します（無料のSupabaseを利用します）。ローカルPCを起動しっぱなしにする必要がない構成にするため、アプリ本体も無料のRenderにデプロイします。

---

## デプロイ手順（無料・PCをつけっぱなしにしなくてOK）

以下はすべて無料のプランで完結します。カード登録が必要になる場合がありますが、料金は発生しません（各サービスの無料枠の範囲内で使う想定です）。

### 1. Supabase（データベース）を用意する

1. https://supabase.com にアクセスし、アカウントを作成してログイン
2. 「New Project」で新しいプロジェクトを作成（プロジェクト名は任意、パスワードは控えておく）
3. 作成後、プロジェクトのトップ画面上部にある **「Connect」** ボタンをクリック（Project Settingsの中ではありません）
4. 表示されるタブ（Framework / Server / Direct / ORM / MCP など、画面のバージョンによって多少異なります）の中から **「Server」** を選ぶ
   - 汎用のバックエンドサーバー向けの接続情報です。今回のNode.js/Expressアプリはこれに該当します
   - 「Direct」はIPv6専用でRenderのような多くのホスティングと相性が悪いため選ばないでください
5. 表示された接続文字列をコピーする（`postgresql://postgres.xxxxx:[YOUR-PASSWORD]@aws-...pooler.supabase.com:5432/postgres` や `:6543/postgres` のような形。ポート番号はどちらでも今回のアプリでは問題なく動作します）
   - `[YOUR-PASSWORD]` の部分を、プロジェクト作成時に設定したパスワードに置き換える
   - これが `DATABASE_URL` になります

> Supabaseの管理画面はレイアウトが更新されることがあります。ここに書いたタブ名と実際の画面表示が違う場合は、「バックエンドサーバー」や「常時起動するアプリ」向けの接続情報を探してください。

> 補足: Supabaseの無料プランは1週間アクセスがないと自動的に一時停止しますが、次にアクセスがあれば約30秒で自動的に復帰し、データは消えません。

### 2. GitHubにコードを置く

1. https://github.com でアカウントを作成（すでにあればスキップ）
2. 新しいリポジトリを作成（Public/Privateどちらでも可）
3. このフォルダ（`kids-reward-app`）の内容をそのリポジトリにpushする

```
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin <あなたのリポジトリのURL>
git push -u origin main
```

### 3. Render（アプリのホスティング）にデプロイする

1. https://render.com にアクセスし、GitHubアカウントでログイン
2. 「New +」→「Web Service」を選択し、先ほどのGitHubリポジトリを選ぶ
3. 以下を設定する
   - **Runtime**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
4. 「Environment」欄で環境変数を追加する
   - `DATABASE_URL` = 手順1でコピーしたSupabaseの接続文字列
5. 「Create Web Service」で作成すると、自動でビルド・起動され、`https://xxxxx.onrender.com` のようなURLが発行されます

このURLに、家でも外出先でも、Wi-Fiに関係なくスマホ・PCからアクセスできます。

> 補足: Renderの無料プランは15分アクセスがないとスリープします。次にアクセスした時、起動に30〜50秒ほどかかることがあります（子どもが少し待つことになります）。

### 4. 動作確認

デプロイ後のURL（`https://xxxxx.onrender.com`）にアクセスし、保護者ページ・こどもページがこれまでと同じように動くことを確認してください。データはSupabase上に保存されるので、Renderがスリープ・再起動してもデータは消えません。

## 保護者用PIN

保護者メニューには簡易PIN（初期値 `0000`）が必要です。「設定」タブから変更できます。
※ これは家庭内での誤操作防止用の簡易的な仕組みで、強固な認証ではありません。

## ローカルでの動作確認（開発用）

デプロイ前にPC上で試したい場合は、`.env.example` を `.env` にコピーし、`DATABASE_URL` にSupabaseの接続文字列を設定してから起動します。

```
npm install
npm start
```
